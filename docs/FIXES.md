# FIXES вАФ Civitics Platform

Actionable improvement backlog. Every item has a priority, complexity, and enough context to hand to Qwen or Claude Code directly.

**Priority key:**
- рЯФі Critical вАФ Bug that breaks or blocks real functionality
- рЯЯ† High вАФ Meaningful product/UX gap, address soon
- рЯЯ° Medium вАФ Worthwhile improvement, schedule when practical
- рЯЯҐ Quick Win вАФ Small effort, high visible impact (batch these)
- вђЬ Future вАФ Phase 2+ or requires significant design/pipeline work

**Complexity key:** S = <2h ¬Ј M = 2вАУ8h ¬Ј L = 1вАУ3 days ¬Ј XL = multi-day + planning

**Workflow:** Every bullet has a stable ID (`<!--id:FIX-NNN-->`). Don't remove or renumber IDs вАФ they're the handle commits reference via `Fixes: FIX-NNN` trailers. Completion state is sourced from `docs/done.log`; regenerate this file's checkboxes with `pnpm fixes:sync`. See [CLAUDE.md](../CLAUDE.md#fixes-workflow) for details.

**Section rules:**
- Active sections вАФ `[x]` items are fine (checked by `fixes:sync`). Periodically move completed clusters to `## COMPLETED` for readability.
- `## COMPLETED` вАФ **`[x]` only**. A `[ ]` item here means it was moved before it shipped вАФ move it back to the active section.
- Deferred / blocked items always stay in active sections as `[ ]`, never in COMPLETED. If a deferred item was closed by a broad "closeout" commit, add a `reopen` line to `done.log` and uncheck it.
