# FIXES вҖ” Civitics Platform

Actionable improvement backlog. Every item has a priority, complexity, and enough context to hand to Qwen or Claude Code directly.

**Priority key:**
- рҹ”ҙ Critical вҖ” Bug that breaks or blocks real functionality
- рҹҹ  High вҖ” Meaningful product/UX gap, address soon
- рҹҹЎ Medium вҖ” Worthwhile improvement, schedule when practical
- рҹҹў Quick Win вҖ” Small effort, high visible impact (batch these)
- в¬ң Future вҖ” Phase 2+ or requires significant design/pipeline work

**Complexity key:** S = <2h В· M = 2вҖ“8h В· L = 1вҖ“3 days В· XL = multi-day + planning

**Workflow:** Every bullet has a stable ID (`<!--id:FIX-NNN-->`). Don't remove or renumber IDs вҖ” they're the handle commits reference via `Fixes: FIX-NNN` trailers. Completion state is sourced from `docs/done.log`; regenerate this file's checkboxes with `pnpm fixes:sync`. See [CLAUDE.md](../CLAUDE.md#fixes-workflow) for details.

**Section rules:**
- Active sections вҖ” `[x]` items are fine (checked by `fixes:sync`). Periodically move completed clusters to `## COMPLETED` for readability.
- `## COMPLETED` вҖ” **`[x]` only**. A `[ ]` item here means it was moved before it shipped вҖ” move it back to the active section.
- Deferred / blocked items always stay in active sections as `[ ]`, never in COMPLETED. If a deferred item was closed by a broad "closeout" commit, add a `reopen` line to `done.log` and uncheck it.
